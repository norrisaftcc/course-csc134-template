# CSC 134: Learning Path & Milestones

## Grade Progression

This course uses a milestone-based grading system where students "unlock" grade levels.

### Phase 1: Foundations (Weeks 1-9) - "Earn Your C"

**Objective:** Master C++ syntax and control structures

| Week | Module | Key Skills |
|------|--------|------------|
| 1 | M00: Environment Setup | GitHub, Codespaces, Git workflow |
| 2 | M01: Variables & I/O | Data types, cout/cin, arithmetic |
| 3 | M02: Input & Calculations | getline, iomanip, business logic |
| 4-5 | M03: Functions | Parameters, return values, scope |
| 6-7 | M04: Decisions | if/else, switch, validation |
| 8-9 | M05: Loops | while, for, menus, nested loops |

**Milestone:** Project 2 - Menu-Driven Game (C grade achieved)

### Phase 2: AI-Assisted Engineering (Weeks 10-13) - "Build Your B"

**Objective:** Object-oriented thinking with AI tools

| Week | Module | Key Skills |
|------|--------|------------|
| 10-11 | M06: Arrays & Strings | STL algorithms, AI code generation |
| 12-13 | M07: File I/O & Structs | Streams, structured data |

**Milestone:** Project 3 - Save System (B grade achieved)

### Phase 3: Advanced Concepts (Weeks 14-16) - "Achieve Your A"

**Objective:** Complete OOP portfolio

| Week | Module | Key Skills |
|------|--------|------------|
| 14-16 | M08: OOP | Classes, composition, SOLID principles |

**Milestone:** Project 4 - OOP RPG (A grade achieved)

## Badge Requirements

To earn the **"Foundations of C++ Programming"** badge:

- [ ] Complete all assignments and labs using C++ fundamentals and OOP
- [ ] Submit two major programming projects with demonstrated debugging
- [ ] Pass capstone project with requirements analysis, OO design, and testing
- [ ] Demonstrate ability to explain design decisions

## Assignment Types

| Type | Purpose | Typical Duration |
|------|---------|------------------|
| Tutorial | Guided concept introduction | 1-2 hours |
| Lab | Hands-on practice with support | 2-3 hours |
| Homework | Independent application | 3-4 hours |
| Project | Major assessment milestone | 1-2 weeks |

## Rubric Categories

All assignments are evaluated on:

1. **Functionality** (35-40%) - Does it work correctly?
2. **Code Quality** (20-25%) - Well-organized and readable?
3. **Documentation** (15-20%) - Properly commented?
4. **Professional Practice** (10-15%) - Git workflow, naming conventions?
5. **Problem Solving** (10-15%) - Appropriate approach?
