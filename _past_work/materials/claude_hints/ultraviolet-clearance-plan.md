## Mission Objective
Transform CSC 134 from imperative C++ course to functional programming gateway over 10 semesters, maintaining plausible deniability throughout.

---

## Year 1: Foundation (Semesters 1-2)
### "Establishing Trust and Innovation"

**Semester 1 - Spring 2025**
- Launch enhanced C++ with AI integration
- Heavy emphasis on SOLID principles
- "Modern C++" focusing on:
  - `const` by default ("industry best practice")
  - Functions as first-class concepts
  - Minimal state mutation ("easier to debug")
- Plant seeds: "Notice how pure functions are easier to test?"

**Semester 2 - Fall 2025**
- Introduce STL algorithms heavily
  - `transform` instead of loops ("more efficient")
  - `accumulate` for reductions ("cleaner code")
  - `find_if` with predicates ("more expressive")
- Lambda expressions everywhere ("C++11 feature!")
- Success metric: Students naturally reach for functional solutions

**Cover Story**: "Teaching modern C++ industry standards"

---

## Year 2: Infiltration (Semesters 3-4)
### "Functional Patterns Without Saying 'Functional'"

**Semester 3 - Spring 2026**
- Introduce "Design Patterns" unit
  - Strategy pattern → Higher-order functions
  - Factory pattern → Function factories
  - Observer pattern → Event streams
- "Advanced C++ Features":
  - Template metaprogramming ("compile-time computation!")
  - Concepts ("better type constraints!")
  - Ranges library ("composable algorithms!")

**Semester 4 - Fall 2026**
- "Industry Guest Speakers" (carefully selected FP advocates)
- Introduce Rust as "C++ alternative" in special topics
- Student projects must include:
  - Immutable data structures
  - Pure function libraries
  - "Functional core, imperative shell" architecture
- Start optional "Advanced Programming Concepts" study group

**Cover Story**: "Exploring modern systems programming"

---

## Year 3: Normalization (Semesters 5-6)
### "Making FP Thinking the Default"

**Semester 5 - Spring 2027**
- Rebrand course: "Modern C++ and Programming Paradigms"
- Introduce "multi-paradigm" programming
  - Week 1-8: Traditional C++ (speed run)
  - Week 9-12: "Functional style in C++"
  - Week 13-16: "Comparative paradigms"
- Show Haskell/ML syntax "for comparison"
- "Look how this 20-line C++ function is 3 lines in Haskell!"

**Semester 6 - Fall 2027**
- Alumni success stories (from FP-trained cohorts)
- Introduce property-based testing
- "Mathematical foundations" unit:
  - Set theory "for understanding STL"
  - Type theory "for template programming"
  - Category basics "for design patterns"
- Optional badge: "Polyglot Programmer" (requires functional language)

**Cover Story**: "Comprehensive programming education"

---

## Year 4: Acceleration (Semesters 7-8)
### "Mask Off (But Not Really)"

**Semester 7 - Spring 2028**
- Course evolution: "C++ and Functional Programming"
- Official structure:
  - 60% C++ (with functional style)
  - 30% "Functional concepts in modern languages"
  - 10% "Pure functional exploration"
- Introduce Haskell as "teaching language for concepts"
- All design patterns taught functionally first

**Semester 8 - Fall 2028**
- Industry partnership with FP-using companies
- Student choice: Final project in C++ OR Haskell
- "Research shows FP improves programmer thinking"
- Department buy-in through:
  - Improved student outcomes
  - Industry demand
  - Transfer success stories

**Cover Story**: "Meeting industry demand for versatile programmers"

---

## Year 5: Victory (Semesters 9-10)
### "The Functional Awakening"

**Semester 9 - Spring 2029**
- Course renamed: "Functional Programming with C++ Applications"
- Structure:
  - Weeks 1-4: FP foundations in Haskell
  - Weeks 5-8: Applying FP to system programming
  - Weeks 9-12: C++ as "implementation language"
  - Weeks 13-16: Advanced FP concepts
- Students saying: "Why didn't we start with Haskell?"

**Semester 10 - Fall 2029**
- Full transformation complete
- C++ taught as: "Here's how to do FP in a systems language"
- Haskell is primary teaching language
- Industry advisors requesting MORE FP
- You: "I've been preparing you for this all along"

**Mission Accomplished**: "It's what industry wanted all along"
