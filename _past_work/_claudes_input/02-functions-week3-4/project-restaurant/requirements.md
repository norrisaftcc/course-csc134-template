# Restaurant Management System - Project Requirements
## Module 02 Project (100 points)

## Overview
Create a fully functional restaurant ordering system that demonstrates your mastery of functions, parameters, and program organization.

## Core Requirements

### 1. Function Organization (25 points)
- **Minimum 6 meaningful functions** (not counting main)
- Each function should have **one clear purpose**
- Use function prototypes
- Organize your code logically

### 2. Parameter Usage (25 points)
- Use both **pass by value** and **pass by reference** appropriately
- At least **2 functions must use reference parameters** to modify values
- Pass multiple parameters to functions
- Show understanding of when to use each type

### 3. Program Functionality (20 points)
- Welcome customer with restaurant branding
- Display a menu with at least 4 items
- Process multiple orders in a session
- Calculate subtotal, tax, and total
- Apply at least one type of discount
- Display a formatted receipt

### 4. User Experience (15 points)
- Professional, themed output
- Clear prompts and instructions
- Input validation (no crashes on bad input)
- Consistent formatting throughout
- Creative restaurant theme

### 5. Code Quality (10 points)
- Meaningful variable and function names
- Proper indentation and formatting
- Comments explaining complex logic
- Constants for prices and rates
- No unnecessary global variables

### 6. Flash Collaboration Log (5 points)
- Submit your Flash conversation showing:
  - How you developed your restaurant theme
  - Problem-solving process for any bugs
  - Creative brainstorming for features

## Getting Started Tips

1. **Start with Flash**: Brainstorm your theme and menu items
2. **Plan your functions**: List what each will do before coding
3. **Build incrementally**: Get basic version working, then add features
4. **Test frequently**: Run after adding each function
5. **Make it yours**: The best projects show personality!
